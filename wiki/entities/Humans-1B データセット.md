# Humans-1B データセット

[[entities/Sapiens2]] の自己教師あり事前学習に用いられた、**約10億枚の高品質人物画像コーパス**。タスクラベルを一切持たず、唯一の制約は「目立つ人物が少なくとも1人写っていること」。前世代 Sapiens の 300M 枚から大幅にスケールした。出典は [[sources/SAPIENS2]]（§3.1）。

## キュレーション手順

web 規模の **約 4B 枚**のプールから、多段フィルタで人物中心コンテンツを抽出する:

1. **多段フィルタ**: bounding box 検出 → head-pose 推定 → 美的（aesthetic）・realism スコアリング → CLIP 特徴 → text-overlay 検出。realism / 品質チェック不合格を除去。
2. **解像度条件**: 短辺で **≥384px** の人物を少なくとも1人含む画像を保持（複数人可）。
3. **重複除去**: perceptual hashing + deep-feature 近傍剪定。
4. **バランス調整**: 視覚埋め込みをクラスタリングし selective sampling（DINOv2 方式）で、ポーズ・視点・遮蔽・服装・シーン種別・照明にわたり内容を均衡化。
5. **較正**: 閾値・バランス上限を小規模な人手監査で調整。

結果として、年齢・民族・背景・実世界条件にわたり多様で、バランスが取れた約 **1B 枚**の高品質人物画像が得られる。

## 設計思想

事前学習では**タスクラベルも人物固有の事前知識も注入しない**。「スケールは、分布が多様・バランス・高品質なときにのみ効く」という立場から、人手監査でバランスを較正した点が特徴。この prior-free なデータ駆動方針が、[[entities/Sapiens2]] の汎化と大規模化を支える。
