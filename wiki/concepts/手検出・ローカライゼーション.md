# 手検出・ローカライゼーション

画像中の手のバウンディングボックスと手側（左右）ラベルを推定するタスク。3D 手姿勢推定（[[concepts/3D手姿勢推定]]）の多くは手領域のタイトクロップ上で動作するため、**in-the-wild への汎化には堅牢な手検出器が前提**となる。それにもかかわらず、人体・顔検出が大きく進展した一方で、手検出は長年ブレークスルーを欠いてきた（[[sources/WiLoR]] §2）。

## 課題

- 既存手法（[[entities/MediaPipe]]、OpenPose）は複数手・難ポーズの検出に失敗しやすい
- ContactHands（Mask R-CNN 拡張）や ViTDet は精度は出るが**リアルタイム動作できない**（ContactHands は 3 FPS、ViTDet は 1 FPS）
- 根本原因は、**多様なオクルージョン・モーションブラーを含む in-the-wild の大規模訓練データの不足**

## WiLoR の手検出パイプライン

[[sources/WiLoR]] は、物体検出の知見を移植したリアルタイム検出器を提案する。

### アーキテクチャ（one-stage backbone-neck-head）
1. **バックボーン**: DarkNet（[70], YOLOv3 系）。最後の 3 つの特徴マップ {C3, C4, C5} を抽出
2. **ネック**: **PANet（Path Aggregation Network）** — FPN（Feature Pyramid Network）を bottom-up パス拡張で拡張し、top-down と bottom-up 両方向でマルチスケール特徴を統合
3. **3 つの検出ヘッド**: 異なるアンカー解像度で bbox・手側ラベル・手関節を予測
4. **anchor-free 設計**（CenterNet [16] に倣う）: 事前定義アンカーに依らず bbox 座標を直接予測

### マルチタスク損失（式5）
$$\mathcal{L} = \lambda_0 \mathcal{L}_{BCE} + \lambda_1 \mathcal{L}_{DFL} + \lambda_2 \mathcal{L}_{CIoU} + \lambda_3 \mathcal{L}_{kpts}$$
- $\mathcal{L}_{BCE}$: box ラベルの二値交差エントロピー
- $\mathcal{L}_{DFL}$: 分布焦点損失（bbox 分布の差、[40]）
- $\mathcal{L}_{CIoU}$: bbox の不一致 [101]
- $\mathcal{L}_{kpts}$: ランドマークの L2 損失。**関節キーポイント監督が検出の精度・頑健性を有意に改善**（RetinaFace [13] の知見）

## 効率と精度

- **Proposed-S**: 175 FPS / 7Mb、**Proposed-M**: 138 FPS / 25Mb（>130 FPS のリアルタイム）
- 従来 SOTA の ContactHands 比で **45× 高速・32× 小型**、mAP 平均 +26%（[[entities/WHIM データセット]] / COCO-WholeBody / Oxford-Hands で評価）
- 軽量さによりモバイル応用や重い 3D パイプラインへの組み込みでもオーバーヘッドが小さい

## なぜ重要か

堅牢な検出は、フレーム単位の 3D 姿勢推定で問題となる**ジッタ（フレーム間のちらつき）を抑え、安定した 4D 再構成**を可能にする。WiLoR は時間モジュールなしで HaMeR を上回る時間的コヒーレンスを達成しており（[[sources/WiLoR]] Table 6）、これは検出の安定性に起因する。
