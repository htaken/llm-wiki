# ウィンドウ関数

集約値を計算しつつ、**元の行を保持したまま**結果に付与できる関数。`OVER (...)` 句で対象範囲（ウィンドウ）を指定する。

## 通常の `GROUP BY` との違い

通常の集約は行が潰れる:

```sql
SELECT user_id, SUM(amount)
FROM orders
GROUP BY user_id;
```

→ user_idごとに1行になる。

ウィンドウ関数は行を残す:

```sql
SELECT
  user_id,
  amount,
  SUM(amount) OVER (PARTITION BY user_id)
FROM orders;
```

orders:

| user_id | amount |
|---|---|
| 1 | 100 |
| 1 | 300 |
| 2 | 500 |

結果:

| user_id | amount | sum |
|---|---|---|
| 1 | 100 | 400 |
| 1 | 300 | 400 |
| 2 | 500 | 500 |

## PARTITION BY

ウィンドウをグループ分けする指定。`GROUP BY` と似ているが、行を潰さない点が違う。

## ROW_NUMBER()

各パーティション内で1, 2, 3... の連番を振る。

```sql
ROW_NUMBER() OVER (
  PARTITION BY user_id
  ORDER BY created_at DESC
)
```

→ ユーザーごとに新しい順で番号を振る。

典型用途: **「各ユーザーの最新1件」を取り出す**（`ROW_NUMBER = 1` でフィルタ）。

## RANK()

順位付け。同値は同順位を共有し、次の順位は飛ぶ。

```sql
RANK() OVER (ORDER BY amount DESC)
```

| amount | rank |
|---|---|
| 100 | 1 |
| 90 | 2 |
| 90 | 2 |
| 80 | 4 |

→ 90が2位タイなので、次は3位ではなく4位になる。

## 集約関数のウィンドウ版

`SUM` 以外にも `AVG`, `COUNT`, `MIN`, `MAX` 等が `OVER` で使える。「全体に対する割合」「移動平均」などの計算で強力。
