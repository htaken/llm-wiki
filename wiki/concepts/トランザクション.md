# トランザクション

複数のSQL操作を **「ひとまとまり」** として扱う仕組み。全成功か全失敗のどちらかしか起きない（中途半端な状態にならない）。

## 銀行送金の例

A口座から1000円減らし、B口座に1000円加える処理を考える:

```sql
UPDATE accounts SET balance = balance - 1000 WHERE id = 1;  -- Aから減算
UPDATE accounts SET balance = balance + 1000 WHERE id = 2;  -- Bへ加算
```

途中で失敗すると、Aから引かれただけでBに加算されない不整合が発生する。

## トランザクションで囲む

```sql
BEGIN;

UPDATE accounts SET balance = balance - 1000 WHERE id = 1;
UPDATE accounts SET balance = balance + 1000 WHERE id = 2;

COMMIT;
```

- 全成功 → `COMMIT` で確定
- 失敗 → `ROLLBACK` で実行前の状態へ戻す

```sql
ROLLBACK;
```

## ACID特性

トランザクションが保証する4つの性質:

| | 意味 |
|---|---|
| **A**tomicity（原子性） | 全成功か全失敗。中途半端な状態を残さない |
| **C**onsistency（一貫性） | 制約等のルールを破らない状態を維持する |
| **I**solation（独立性） | 同時実行されるトランザクションが互いに干渉しない |
| **D**urability（永続性） | COMMITされた変更は障害があっても失われない |

Consistencyは [[concepts/SQL制約]] と組み合わせて初めて意味を持つ（制約があるからこそ「一貫した状態」を定義できる）。
