# LLM Wiki スキーマ

このファイルはWikiの構造・規約・ワークフローを定義する。LLMはこのファイルに従ってWikiを管理する。

## ディレクトリ構造

- `raw/` — ソースドキュメント（読み取り専用。LLMは絶対に変更しない）
- `raw/assets/` — 画像等の添付ファイル
- `wiki/sources/` — ソース要約ページ
- `wiki/entities/` — エンティティページ（人物、ツール、ライブラリ、組織、プロジェクト等）
- `wiki/concepts/` — コンセプトページ（技術概念、理論、手法等）
- `wiki/index.md` — 全ページカタログ
- `wiki/log.md` — 操作ログ

## ページ規約

### フロントマター

すべてのWikiページにYAMLフロントマターを付与する:

```yaml
---
title: ページタイトル
type: source | entity | concept
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [タグ1, タグ2]
sources: [ソースファイル名]  # このページの根拠となるソース
aliases: [別名1, 別名2]     # Obsidianのエイリアス（任意）
---
```

### リンク形式

- Obsidian形式 `[[ページ名]]` を使用する
- 別ディレクトリのページへは `[[entities/ページ名]]` のようにパスを含める
- エイリアス表示が必要な場合は `[[ページ名|表示名]]` を使用する

### ファイル名

- 日本語のページ名をそのままファイル名にする（例: `トランスフォーマー.md`）
- 英語の固有名詞はそのまま使う（例: `PyTorch.md`）
- スペースはそのまま使用する（Obsidianが対応している）

### 言語

- Wikiの記述は日本語
- 英語ソースの場合も要約・解説は日本語で書く
- 固有名詞・技術用語は原語のまま使って良い

## ワークフロー

### Ingest（ソース取り込み）

1. ソースを読み、ユーザーと主要な内容を確認する
2. `wiki/sources/` にソース要約ページを作成する
   - タイトル、著者、URL/出典、要約、キーポイント、引用を含める
3. ソースに登場するエンティティ・コンセプトについて:
   - 既存ページがあれば → 新しい情報を追記し、ソースへの参照を追加する
   - 既存ページがなければ → 新規ページを作成する
   - 既存の記述と矛盾する場合 → 矛盾を明記し、両方のソースを引用する
4. 関連する既存ページにクロスリファレンス（`[[リンク]]`）を追加する
5. `wiki/index.md` を更新する
6. `wiki/log.md` に記録を追加する

### Query（質問応答）

1. `wiki/index.md` を読み、関連ページを特定する
2. 関連ページを読み込み、回答を生成する
3. 回答にはWikiページへの参照を含める
4. 有用な回答（比較分析、合成的考察等）はユーザーと相談の上、Wikiに新規ページとして保存する
5. 必要に応じて `wiki/log.md` に記録を追加する

### Lint（健全性チェック）

以下を検出し、報告する:
- ページ間の矛盾
- 新しいソースで更新された古い情報
- 孤立ページ（他ページからリンクされていない）
- 言及されているがページが存在しないコンセプト/エンティティ
- 欠落しているクロスリファレンス
- 調査すべきデータギャップ

## index.md の形式

カテゴリ別にページを一覧する。各エントリはリンクと一行要約:

```markdown
## ソース
- [[sources/ページ名]] — 一行要約

## エンティティ
- [[entities/ページ名]] — 一行要約

## コンセプト
- [[concepts/ページ名]] — 一行要約
```

## log.md の形式

時系列の操作ログ。各エントリは以下の形式:

```markdown
## [YYYY-MM-DD] 操作種別 | 対象

操作の概要。作成・更新したページの一覧。
```

操作種別: `ingest`, `query`, `lint`, `update`
