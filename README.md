# LLM Wiki

LLMが構築・管理するパーソナルナレッジベース。

研究活動や開発活動で得た知識を、ソースの取り込みを通じて構造化されたWikiとして蓄積していく。人間がソースを選び問いを立て、LLMが要約・整理・相互参照の維持を担う。

## 構造

```
llm-wiki/
├── CLAUDE.md              # スキーマ（運用ルール・構造定義）
├── raw/                   # ソースドキュメント（不変）
│   └── assets/            # 画像等の添付ファイル
└── wiki/                  # LLMが管理するWiki本体
    ├── index.md           # 全ページカタログ
    ├── log.md             # 操作ログ（時系列）
    ├── sources/           # ソース要約ページ
    ├── entities/          # エンティティページ（人物・ツール・組織等）
    └── concepts/          # コンセプトページ（技術概念・手法等）
```

## 3層アーキテクチャ

| レイヤー        | 役割                         | 編集者      |
| ----------- | -------------------------- | -------- |
| `raw/`      | 生のソースドキュメント。記事、論文、画像など。不変  | 人間       |
| `wiki/`     | LLMが生成・管理するMarkdownファイル群   | LLM      |
| `CLAUDE.md` | Wikiの構造・規約・ワークフローを定義するスキーマ | 人間 + LLM |

## 操作

- **Ingest** — ソースを取り込み、要約ページ作成、既存ページ更新、クロスリファレンス追加
- **Query** — Wikiに対して質問し、回答を得る。有用な回答はWikiに保存可能
- **Lint** — Wikiの健全性チェック（矛盾、孤立ページ、欠落リンク等の検出）

## 使い方

1. `raw/` にソースを配置する
2. Claude Codeに取り込みを指示する（例: `raw/論文.md を ingest して`）
3. Obsidianで結果を確認する
4. 質問や探索を通じてWikiを育てていく

## ツール

- **Obsidian** — Wikiのビューア・ブラウザ。`[[リンク]]` やグラフビューで構造を俯瞰できる
- **Obsidian Web Clipper** — Web記事をMarkdownに変換してrawに取り込む
- **Claude Code** — Wikiの構築・管理を担うLLMエージェント
